Manual

Overview

This page is used to inspect ESP32 sensor clients, update their configuration, and monitor the latest reported battery voltage and sensor readings. The table combines live data from the server log with configuration values read from each client record.

Basic Workflow

  • Press Full Sync with ESP32 to fetch the current client list and the current stored settings from the server.
  • Edit any fields in the table.
  • Press WRITE MODIFIED TO SERVER to send only changed values to the ESP32 server.
  • Wait for the client to report again. Some changes take effect only after the client receives the updated config from the server.

Field Guide

  • MAC Address: Unique identifier of the ESP32 sensor client.
  • Name: Short label stored on the server for easier identification in logs and the web UI.
  • Sleep (s): Interval between measurement cycles. Valid values: 0 to 86400. Between measurements, the device goes to deep sleep. If 0 is specified, the device does not go to deep sleep, stays awake, and performs measurements every 1 second.
  • Work Delay (ms): Delay after the client initializes the I2C bus and before it starts sensor access. Valid values: 1 to 65535.
  • Batt Avg: Number of ADC samples averaged for the battery-voltage reading. Valid values: 1 to 255.
  • SHT Avg: Number of SHT40 measurement cycles averaged for temperature and humidity. Valid values: 1 to 255.
  • SHT Wait (ms): Delay between sending the SHT40 measurement command and reading the result. Valid values: 1 to 65535.
  • DPS OSR: DPS310 oversampling setting used by the pressure readout. The code-to-oversampling mapping is 0=x1, 1=x2, 2=x4, 3=x8, 4=x16, 5=x32, 6=x64, 7=x128.
  • DPS Avg: Number of DPS310 pressure/temperature measurement cycles averaged. Valid values: 1 to 255.
  • DPS Wait (ms): Delay between sending each DPS310 measurement command and reading the result. The current client code uses this same wait after both DPS310 commands in the sequence. Valid values: 1 to 65535.
  • Hash: Configuration hash used to detect whether the client and server are using the same config.

Live Data

The live data columns show the most recent values received by the web server from each client. These values update automatically while the page is open. A configuration change is not confirmed until the client reports again and the displayed hash matches the server-side configuration.

Telegraf Logging

When logging is enabled for a client, the web server forwards incoming measurements to the configured Telegraf destination. The bucket name is attached as a tag in the exported line protocol.

Notes and Cautions

  • Very short sensor wait times can produce invalid or stale readings if the sensor conversion has not finished yet.
  • Changing DPS310 oversampling may require a different DPS wait time for stable measurements.
  • If values appear to revert after editing, run a full sync and verify that both the server firmware and client firmware are updated to the same config schema.
